Instalação do Agent Datadog em AWS Lambda (Serverless)

Projeto: aws-serverless-datadog
Runtime: nodejs22.x
Região: us-east-1
Tags: env:dev service:aws-serverless-datadog version:1.0.0

1. Pré-requisitos

  • Node.js 18+
  • AWS CLI configurado com profile danilo-profile
  • Serverless Framework v3
  • Conta Datadog com API Key e APP Key

2. Variáveis de Ambiente

export DD_API_KEY="sua-api-key"
export DD_APP_KEY="sua-app-key"

3. Layers da Datadog (Lambda)

O agent Datadog em Lambda funciona através de Layers — não há instalação de pacote. São duas layers:

LayerARNFunção
Datadog Extension arn:aws:lambda:us-east-1:464622532012:layer:Datadog-Extension:65 Coleta e envia logs, métricas e traces via HTTP para o Datadog
Datadog Tracer (Node.js 22) arn:aws:lambda:us-east-1:464622532012:layer:Datadog-Node22-x:113 Instrumentação APM automática (dd-trace + datadog-lambda-js)

4. Configuração no serverless.yml

provider:
  runtime: nodejs22.x
  environment:
    # Unified Service Tagging
    DD_ENV: dev
    DD_SERVICE: aws-serverless-datadog
    DD_VERSION: '1.0.0'
    # APM
    DD_TRACE_ENABLED: 'true'
    DD_LAMBDA_HANDLER: src/handler.hello
    # Logs
    DD_SERVERLESS_LOGS_ENABLED: 'true'
    DD_API_KEY: ${env:DD_API_KEY}

functions:
  hello:
    handler: /opt/nodejs/node_modules/datadog-lambda-js/handler.handler
    layers:
      - arn:aws:lambda:us-east-1:464622532012:layer:Datadog-Extension:65
      - arn:aws:lambda:us-east-1:464622532012:layer:Datadog-Node22-x:113

5. Como Funciona o Wrapper

  1. O handler da função aponta para o wrapper da Datadog Layer: /opt/nodejs/node_modules/datadog-lambda-js/handler.handler
  2. A variável DD_LAMBDA_HANDLER indica o handler real da aplicação: src/handler.hello
  3. O wrapper inicializa o tracer, executa o handler real e coleta spans automaticamente

6. Deploy

npm install
npx serverless deploy --stage dev

7. Validação

# Testar endpoint
curl https://x6fpcyfn4h.execute-api.us-east-1.amazonaws.com/dev/hello

# Verificar traces

8. Arquitetura de Observabilidade

Lambda Handler
  ├── dd-trace (APM — injeta trace_id/span_id nos logs)
  ├── hot-shots → DogStatsD (porta 8125) → Extension → Datadog
  └── stdout JSON → Extension (DD_SERVERLESS_LOGS_ENABLED) → Datadog

Sem Forwarder. A Extension envia tudo via HTTP.
